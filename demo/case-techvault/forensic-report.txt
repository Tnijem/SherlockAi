EXECUTIVE SUMMARY

CyberTrace Forensics was retained to conduct a forensic examination of (1) Kevin Zhang's TechVault-issued laptop (Dell Precision 5570, Asset Tag TV-ENG-2847), (2) TechVault's source code repository access logs, (3) TechVault's email server logs, and (4) network egress logs. This report details our findings regarding data exfiltration activities during the period November 1 - December 31, 2025.

METHODOLOGY

All examinations were conducted using industry-standard forensic tools:
- EnCase Forensic v23.2 for disk imaging and analysis
- Axiom Cyber v8.1 for artifact recovery
- Splunk Enterprise for log analysis
- Custom Python scripts for log correlation

Chain of custody was maintained throughout. All evidence was hashed (SHA-256) at acquisition.

FINDINGS

1. SOURCE CODE REPOSITORY ACCESS LOGS

TechVault uses GitLab Enterprise (self-hosted, air-gapped) for source code management. Zhang's account (kzhang@techvault.com) had access to the following repositories:

  - vaultlock-core (main encryption engine)
  - vaultlock-kda7 (key derivation algorithm — RESTRICTED)
  - vaultlock-api (customer-facing API)
  - vaultlock-docs (internal architecture documentation)
  - vaultlock-benchmarks (performance testing and results)

ANOMALOUS ACTIVITY:

November 18, 2025 (Monday), 11:14 PM - 2:47 AM (after hours):
  - git clone of vaultlock-core: 2,134 files, 847 MB
  - git clone of vaultlock-kda7: 423 files, 156 MB
  - git clone of vaultlock-api: 891 files, 234 MB
  - git clone of vaultlock-docs: 399 files, 89 MB
  - TOTAL: 3,847 files, 1.33 GB

  CONTEXT: Zhang's normal Git activity averaged 15-25 file operations per day.
  This represents a 15,000% increase over normal activity.
  Zhang had not cloned the full vaultlock-kda7 repository since initial onboarding in 2022.

November 22, 2025 (Friday), 6:30 PM:
  - Accessed vaultlock-docs/customers/ directory
  - Downloaded deployment guides for: Meridian Bank, Apex Financial, Sterling Insurance,
    Pacific Health Partners, Quantum Dynamics, and 10 additional enterprise clients
  - These documents contain customer-specific configuration details, API keys (encrypted),
    and integration architectures

December 15, 2025 (Monday), 9:15 AM:
  - Accessed vaultlock-kda7/spec/KDA7-SPECIFICATION-v4.2.pdf
  - This document was last accessed by Zhang on June 3, 2025 (6+ months prior)
  - The document is the complete mathematical specification of the KDA-7 algorithm
  - Only 4 employees at TechVault have access to this file

2. USB DEVICE CONNECTION

December 2, 2025 (Tuesday), 12:34 PM:
  Windows Event Log (ID 2003, 2100, 2101) records connection of:
  - Device: Samsung T7 Shield Portable SSD, 2TB
  - Serial: S6XNNS0WB00483K
  - Drive letter assigned: E:\

  File system journal analysis shows:
  - 1.33 GB of data written to E:\ at 12:34-12:41 PM
  - Folder structure created: E:\vault\, E:\vault\core\, E:\vault\kda7\, E:\vault\api\, E:\vault\docs\

  DLP (Data Loss Prevention) software was installed but set to "monitor only" mode

CONCLUSIONS

1. Zhang engaged in systematic data exfiltration over a 6-week period.
2. The exfiltrated data includes the complete VaultLock source code, the KDA-7 specification, customer deployment data, and technical documentation.
3. The timing of Zhang's job application (pre-resignation) and the exfiltration activities strongly suggest premeditated theft.
4. Zhang's use of a secure deletion tool and clearing of browser history/Recycle Bin indicates awareness that his activities were improper and an attempt to conceal evidence.
5. Zhang's exit interview certification was false — data had been copied to a personal USB device and forwarded to a personal email account.
