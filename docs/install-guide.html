2
Install macOS
20–40 min
Skip this section if macOS is already installed and you're setting up a machine that's ready to use.
Step 2.1 — Boot into macOS Recovery

Power on the Mac. Immediately hold ⌘ Command + R until the Apple logo appears. This boots into macOS Recovery mode.

For Apple Silicon Macs: hold the Power button until "Loading startup options" appears, then click Options.

Step 2.2 — Erase the Disk (Fresh Install Only)

In Recovery, open Disk Utility. Select the internal disk (usually "Macintosh HD"), click Erase, and format as APFS. Close Disk Utility.

This permanently erases everything on the drive. Only do this on a new or repurposed machine.
Step 2.3 — Reinstall macOS

Back in Recovery, choose Reinstall macOS and follow the prompts. The machine needs internet for this step — connect ethernet or select Wi-Fi when prompted.

When the installer asks you to select a disk, choose "Macintosh HD".

Step 2.4 — Initial macOS Setup

After install, the Mac will restart and show the Setup Assistant.

  • Country / Language: select appropriately
  • Wi-Fi: connect or skip (use ethernet)
  • Apple ID: Skip — click "Set Up Later" (not required for Sherlock)
  • Account name: use something like sherlock or the firm name
  • Password: pick a strong local password and record it
  • Analytics / Siri: turn off both
  • Screen Time: skip
3
System Setup
5 min
Step 3.1 — Open Terminal

Press ⌘ Space, type Terminal, press Enter.

Keep this window open for the rest of the install.

Step 3.2 — Disable Sleep

Prevents the machine from going to sleep during long downloads.

$sudo systemsetup -setcomputersleep Never
Step 3.3 — Set Computer Name

Replace SherlocK-FirmName with the law firm's name. No spaces.

$sudo scutil --set ComputerName "Sherlock-FirmName"
$sudo scutil --set LocalHostName "Sherlock-FirmName"
Step 3.4 — Install Xcode Command Line Tools

Required for Homebrew and Python. This will show a dialog — click Install.

$xcode-select --install
Wait for the install to complete before continuing (2–5 min).
4
Install Prerequisites
10–20 min
Step 4.1 — Install Homebrew

Homebrew is macOS's package manager. Paste this into Terminal:

$/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

When it finishes, it will print two commands to add Homebrew to your PATH. Run them — they look like:

$echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile && eval "$(/opt/homebrew/bin/brew shellenv)"
Step 4.2 — Install Python 3
$brew install python@3.11
Step 4.3 — Install Docker Desktop

Docker runs the Ollama AI engine and ChromaDB database in containers.

$brew install --cask docker

After it installs, open Docker Desktop from your Applications folder and complete its one-time setup. Wait until the Docker icon in the menu bar shows "Docker Desktop is running".

Docker must be running before you run the Sherlock installer. Check the menu bar icon.
Step 4.4 — Install Optional Tools (Recommended)

Enables OCR on image scans and support for legacy Word/Excel files:

$brew install tesseract && brew install --cask libreoffice
5
Copy Sherlock from USB
2–5 min
Step 5.1 — Plug in the Ventoy USB Drive

Insert the USB. It will appear in Finder as Ventoy or a similar name. If macOS asks what to do with it, click Ignore.

Step 5.2 — Find the USB mount point
$ls /Volumes/

Look for a volume named Ventoy or SHERLOCK in the output.

Step 5.3 — Extract Sherlock

Replace Ventoy below if your USB has a different name:

$cp /Volumes/Ventoy/sherlock-source.tar.gz ~/sherlock-source.tar.gz
$cd ~ && tar -xzf sherlock-source.tar.gz
$ls ~/Sherlock/
You should see folders like web/, scripts/, and files like setup.sh and docker-compose.yaml.
Step 5.4 — Eject the USB
$diskutil eject /Volumes/Ventoy
6
Run the Installer
20–45 min
The installer is interactive — it will ask a few questions, then do the rest automatically. The longest step is downloading AI models (~4–8 GB total).
Step 6.1 — Navigate to Sherlock
$cd ~/Sherlock
Step 6.2 — Make the installer executable
$chmod +x setup.sh
Step 6.3 — Run the installer
$./setup.sh
Step 6.4 — Answer the Setup Questions

The installer will prompt you for a few settings:

QuestionWhat to Enter
System nameFirm's name, e.g. Smith & Associates
NAS path(s)Leave blank for now — configure later if they have a NAS
Outputs directoryPress Enter to accept default
Uploads directoryPress Enter to accept default
Ollama URLPress Enter to accept default (http://localhost:11434)
JWT expiry hoursPress Enter for default (8 hours)
Whisper modelSee below ↓
Whisper model (Standard / 32GB): enter medium for best accuracy.
Whisper model (Mid / 16GB): enter small — good balance of speed and accuracy.
Whisper model (Compact / 8GB): enter base — keeps memory usage low.
Step 6.5 — Set the AI Model for Your Hardware

After the installer finishes, set the LLM model size appropriate for this machine:

Standard (32GB) — uses the full Sherlock RAG model:

$docker exec sherlock-ollama ollama pull mxbai-embed-large

Mid (16GB) — uses mistral 7B, excellent quality at half the memory:

$docker exec sherlock-ollama ollama pull mistral:7b
$docker exec sherlock-ollama ollama pull mxbai-embed-large
$sed -i '' 's/LLM_MODEL=.*/LLM_MODEL=mistral:7b/' ~/Sherlock/sherlock.conf

Compact (8GB) — uses llama3.2 3B, lightweight and fast:

$docker exec sherlock-ollama ollama pull llama3.2:3b
$docker exec sherlock-ollama ollama pull mxbai-embed-large
$sed -i '' 's/LLM_MODEL=.*/LLM_MODEL=llama3.2:3b/' ~/Sherlock/sherlock.conf
On 8GB machines, close all other apps while Sherlock is running to free up memory.
Step 6.6 — Create the Admin Account

The installer will prompt you to create the first admin login. Enter a username and password for the firm's administrator — write these down.

7
First Login
5 min
Step 7.1 — Open Sherlock in the Browser

Open Safari or Chrome and navigate to:

↗http://localhost:3000
Step 7.2 — Log In

Use the admin username and password you just created. You should see the Sherlock dashboard.

Step 7.3 — Create User Accounts

Go to Admin → Users and create accounts for each attorney or staff member who will use Sherlock.

Step 7.4 — Bookmark it

Drag the URL to the bookmarks bar or set it as the browser homepage for easy access.

Sherlock starts automatically when the computer turns on — users just open the browser and go to localhost:3000.
8
Connect to the Firm's Network
10–20 min
This section lets other computers on the office network access Sherlock without being at the Sherlock machine.
Step 8.1 — Find the Sherlock Machine's IP Address
$ipconfig getifaddr en0

This will print something like 192.168.1.42. Write this down.

Step 8.2 — Set a Static IP (Recommended)

Log into the office router and assign the Sherlock machine a permanent IP address (DHCP reservation). This prevents the address from changing.

The router's admin page is usually at 192.168.1.1 or 192.168.0.1. Look for "DHCP Reservation" or "Static Lease" settings. Use the MAC address printed when you run:

$ifconfig en0 | grep ether
Step 8.3 — Access from Other Office Computers

On any computer in the office, open a browser and go to:

↗http://192.168.1.42:3000

(Replace 192.168.1.42 with the actual IP from Step 8.1)

Step 8.4 — Connect NAS Shared Drives (If Applicable)

If the firm has a NAS (network storage), mount it on the Sherlock machine so it can index those files. In Finder, press ⌘K and enter the NAS address (e.g. smb://192.168.1.10/LegalFiles).

After mounting, note the path it appears at and add it to Sherlock:

$ls /Volumes/

Then in the Sherlock Admin panel → Settings → add the NAS path (e.g. /Volumes/LegalFiles).

9
Verify Everything Works
5 min
Step 9.1 — Run the Status Check
$cd ~/Sherlock && ./setup.sh --status

All items should show ✓. If any show ✗, see Section 10.

Step 9.2 — Test a Search

Log into Sherlock, upload a sample PDF (any legal document), and ask it a question about the document. If you get a response, everything is working.

Step 9.3 — Final Checklist
  • ./setup.sh --status shows all ✓
  • Browser opens http://localhost:3000
  • Admin can log in
  • Test document uploaded and searchable
  • Other office computers can reach Sherlock at its IP:3000
  • Admin account password recorded and stored securely
  • NAS path configured (if applicable)

Troubleshooting
ProblemFix
"Docker not found" Open Docker Desktop from Applications and wait for the menu bar icon to say "running", then re-run ./setup.sh
Sherlock web not responding Run: tail -f ~/Sherlock/logs/sherlock-web.log and look for errors
Ollama not responding Run: docker ps — if sherlock-ollama is missing, run cd ~/Sherlock && docker compose up -d
Out of memory / slow responses Switch to a smaller model (see Section 6.5). Close other apps. Restart Sherlock: cd ~/Sherlock && ./restart.sh
Can't reach from other computers Check firewall: System Settings → Network → Firewall → add Sherlock. Verify IP address hasn't changed.
Models failed to download Ensure internet is connected, then: docker exec sherlock-ollama ollama pull mxbai-embed-large
Forgot admin password Run: cd ~/Sherlock/web && ../venv/bin/python create_admin.py reset
Restart All Services
$cd ~/Sherlock && ./restart.sh
View Live Logs
$tail -f ~/Sherlock/logs/sherlock-web.log
